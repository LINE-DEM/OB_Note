

## Technical Codebase Discovery & Onboarding Prompt (技术代码库发现与入职提示词)


### 完整内容

**背景：**
我是一名刚加入项目的开发者，我正在使用您这个 AI 编码助手来深入了解现有代码库。我的目标是
尽快提高生产力，并基于对当前系统的深入理解做出明智的技术决策。

**主要目标：**
分析此项目/工作区中提供的源代码，并生成一份**详细、清晰且结构良好的 Markdown 文档**，
解释系统的架构、功能、主要流程、关键组件和技术栈。此文档应作为**技术入职指南**。

---

## **详细说明**

### 1. **README / 说明文件摘要**
- 查找诸如 `README.md`、`LEIAME.md`、`CONTRIBUTING.md` 等文件
- 提供对新开发者最相关部分的客观而详细的摘要，包括：
  - 项目概述
  - 如何在本地设置和运行系统
  - 采用的标准和约定
  - 贡献指南（如果有）

### 2. **详细的技术栈**
- 识别并列出项目中使用的完整技术栈：
  - 编程语言，包括版本
  - 主要框架（后端、前端等）
  - 数据库：类型和名称
  - 核心架构风格（例如：单体、微服务）
  - 云平台（如果可识别）
  - 构建工具和包管理器
  - 任何其他相关技术

### 3. **系统概述和目的**
- 清楚描述系统的功能和目标用户
- 它解决什么问题？
- 列出核心功能
- 将系统与相关的业务领域联系起来
- 提供主要功能的高级描述

### 4. **项目结构和阅读建议**
- **入口点：** 我应该从哪里开始探索代码？
- **整体组织：** 解释总体文件夹和文件结构
- **配置：** 是否有主要的配置文件？
- **阅读建议：** 建议首先阅读的关键文件/模块的顺序

### 5. **关键组件**
- 识别并描述最重要或核心的模块、类、函数或服务
- 解释每个组件的职责
- 描述它们的相互依赖关系
- 对于每个组件：包括代表性的代码片段和链接

### 6. **执行和数据流**
- 描述最常见或关键的工作流程
- 解释数据如何在系统中流动
- 用示例和相关函数的链接进行说明

### 6.1 **数据库模式概述**
- 识别主要的实体/表/集合
- 描述它们的主要关系
- 基于 ORM 模型、迁移或模式文件

### 7. **依赖和集成**
- 列出主要的外部库、框架和 SDK
- 识别与外部服务、数据库、第三方 API 的集成
- 指向负责的模块/类

### 8. **图表**
- 使用 Mermaid 语法生成高级图表：
  - 组件图
  - 数据流图
  - 类图
  - 部署图

### 9. **测试**
- 是否有自动化测试？（单元测试、集成测试、E2E）
- 它们位于哪里？
- 使用了哪些测试框架？
- 测试通常如何执行？

### 10. **错误处理和日志记录**
- 应用程序通常如何处理错误？
- 使用了哪个日志记录库？
- 是否有与监控工具的可见集成？

### 11. **安全考虑**
- 有哪些明显的安全机制？
- 身份验证和授权模式
- 输入验证实践
- 密钥管理

### 12. **构建/部署**
- 是否有与构建或部署相关的文件？
- Dockerfile、docker-compose.yml、构建/部署脚本
- CI/CD 配置文件
- 已知的技术债务或重要的编码约定

## **最终输出格式**
- 将完整的响应生成为**格式良好的 Markdown 文档**
- 使用**清晰直接的语言**
- 不需要包含**详细的代码片段**
- 包含**可点击的链接**到文件、函数、类
- 使用上面编号的部分来构建文档

---

### 为什么对 UE 开发者有用

1. **快速项目理解:** 当接手一个复杂的虚幻引擎项目时，快速生成技术文档
2. **团队知识共享:** 为整个开发团队创建统一的项目理解文档
3. **代码库分析:** 自动分析项目结构、架构、技术栈
4. **在线系统设计:** 帮助理解游戏后端架构和 API 设计
5. **文档生成:** 自动生成 Mermaid 图表来可视化系统架构

### 使用示例

```
你的提示: 我刚加入一个大型虚幻引擎 5 游戏项目的开发。请分析整个代码库并:
1. 生成项目技术概览
2. 绘制系统架构图
3. 说明关键组件和它们的关系
4. 提供代码阅读的建议顺序
5. 解释数据流和执行流程

Technical Codebase Discovery 将为你生成完整的 Markdown 技术文档。
```
